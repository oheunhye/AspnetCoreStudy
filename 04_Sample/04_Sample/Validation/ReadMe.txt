유효성 검사 컨트롤
컨트롤에 사용자가 브라우저 통해 데이터를 입력할 경우, 그 값이 유효한 값인지를 확인하는 컨트롤.
유효하지 않은 값을 입력할 경우 사용자에게 오류메세지를 보여줌.

-유효성 검사 컨트롤의 구성
CompareValidator			:사용자가 입력한 값과 다른 컨트롤의 값이 동일한지 비교
CustomValidator				:직접 식을 만들어서 사용자가 입력한 값이 유효한지 체크
RangeValidator				:사용자가 입력한 값이 특정 범위에 속하는 값인지를 체크
RegularExpressionValidator	:사용자가 입력한 값과 정규표현식 비교하여 데이터 입력범위, 패턴일치여부를 체크
RequiredFieldVaildator		:필수항목에 사용자가 값을 입력했는지 혹은 선택작업을 했는지 체크
VaildationSummary			:에러가 있는 경우 사용자에게 선택적으로 메세지 박스를 출력

-컨트롤의 속성
ControlToValidate	:검사할 컨트롤을 지정
EnableClientScript	:클라이언트에서 자바스크립트로 유효성검사를 하겠다는 의미(False:서버에서 유효성검사)
ErrorMessage		:입력값이 유효하지 않을 경우 표시할 메세지
Display				:유효성 검사 컨트롤 표시방법
	1.None	:오류메세지 표시하지 않음
	2.Static:항상 오류메세지 표시할 공간 차지
	3.Dynamic:표시할 오류메세지 있을 때만 공간 차지

-정규식 사용한 유효성 체크(RegularExpressionValidator)
데이터 입력에서 이메일은 반드시 @가 포함, 아이디와 비밀번호는 공백을 포함하지 않음
등 과 같이 특별한 문자를 제어할 경우 정규식을 이용.
\w		:영문 소문자, 대문자, 숫자와 같은 모든 문자를 의미 => [A-Za-z0-9]와 같음
{n,m}	:n번 이상 m번 이하 반복입력
[nmop]	:n,m,o,p중 한 문자 입력
[^nm]	:nm문자를 제외한 나머지 문자
.		:한 문자를 의미 (예.n,m과 같은 한 문자)
*		:없거나 1번 이상 반복되어 입력
\s		:띄어쓰기, 탭 등을 의미
