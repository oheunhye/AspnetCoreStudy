(1)page객체의 Life cycle
클라이언트가 특정 페이지를 요청하면 실행기에 의해 page와 같은 인스턴스를 생성하고 
각 단계별로 처리하여 결과물을 클라이언트에게 반환

1페이지 요청 및 실행
:Aspx파일을 찾아서 해당 page의 인스턴스 생성

2 Init 이벤트 
:페이지 객체 생성된 후 각 컨트롤 속성 초기화 작업에 이용

3 뷰스테이트 복구 ->포스트백 요청시 발생
:이전요청시 페이지 상태를 저장한 뷰스테이트를 이용하여 서버컨트롤 상태를 복구

4 다시 포스트된 데이터 ->포스트백 요청시 발생
:서버컨트롤 폼 필드로부터 반환된 데이터, 컨트롤의 값 등을 처리

5 Page Load이벤트 
:페이지 객체가 메모리에 로드되고 난 후에 발생.
컨트롤에 대한 속성을 변경작업, 클라이언트의 데이터 반영작업

6 포스트데이터 변경이벤트 
:포스트백에서 컨드롤 상태변경에 대한 이벤트 처리
기존의 내용을 고쳤을 시

7 포스트백 이벤트 
:포스트백 요청시(버튼 등) 포스트백 일으킨 서버 컨트롤의 이벤트 작업 처리

8 PreRender이벤트
:페이지 객체를 렌더링하기 전 단계로 오류 해결을 위해 만든 작업단계(디버깅)

9 ViewState
:페이지 객체와 그 내부에 있는 컨트롤의 내용을 뷰스테이트에 저장
컨트롤의 속성값이 저장

10 Render
페이지 객체와 컨트롤들을 HTML로 변환시켜 생성하는 작업

11 Page객체소멸
:HTML로 클라이언트에 표시한 후 해당 페이지 객체를 소멸시키는 작업
UnLoad이벤트에서 진행

----------------------------------------------------------------------------------------

-포스트백
기존에 있던 화면의 내용을 보여줘
이전화면 그대로 있음. ->뷰스테이트에 저장되어 있으므로

-ViewStatus
웹서버와 웹 브라우저 간에 데이터를 요청/응답하기위해 페이지의 상태를 저장하는 
커다란 서버상의 공간을 의미

-Page 클래스의 속성
	속성 : 기능
	Request : HTTP요청 정보에 대한 Request객체를 지칭
	Response: 현재 페이지 요청에 대한 서버의 액세스를 제공
	Sever	: 웹 프로그래밍에서 필요한 유틸리티 기능을 모아둔 객체
	Application: 프로그램수준의 방법과 모든 세션에 대한 이벤트에 대한 액세스를 제공
				->여러명이 프로젝트에 들어와서 정보를 교환가능하게 함
	Session	: 현재 사용자 세션에 대한 정보를 제공
